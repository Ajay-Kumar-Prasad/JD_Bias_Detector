# Evaluation Report — JD Bias Detector

> This file is auto-generated by `training/evaluate.py` after training.
> Run the command below to populate it with real metrics:

```bash
python -m training.evaluate \
    --model_dir models/deberta-jd-bias-v1 \
    --test_data data/annotated/test.jsonl \
    --output    docs/evaluation_report.md
```

---

## Expected output structure

After running evaluate.py, this file will contain:

- Overall macro F1, Precision, Recall
- Inter-annotator agreement (Cohen's κ)
- Per-category bias coverage (% of known biased tokens caught)
- Full seqeval classification report
- Key findings and failure mode notes

---

## data/annotated/ — populated by data_prep.py

The `train.jsonl`, `val.jsonl`, and `test.jsonl` files are generated by:

```bash
python -m training.data_prep --output_dir data/annotated --n_synthetic 1000
```

They are intentionally excluded from version control (see `.gitignore`).
